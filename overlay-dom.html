General

Root Element

All overlays are Single Page Applications rendered into #root. The commonly assumed #application does not exist.

/* Always start with this reset */
html, body, #root { background: transparent !important; }
OBS note: Without this reset, the browser source renders a solid black or dark-grey box instead of a transparent overlay.

Inline Styles

Joystick's JavaScript injects style="color: rgb(...)" and style="width: X%" directly onto elements. Standard CSS rules lose to inline styles — always use !important.

Specificity tip: For stubborn inline styles, use attribute selectors like div.scrollable.text-3xl[style] to maximise specificity.

ElementSelectorNotes
Message container .chat-message Wrap background, border, padding here. Add position: relative; z-index: 1; overflow: visible to prevent Joystick's entry animation pushing it behind other elements.
Username badge .username.username span Use display: inline-block !important to prevent collapsing in flex context. Background gradients and colours go here.
Message text .content Font and text colour for the message body.

  background: #00d2ff !important;
  color: #000 !important;

  border-radius: 6px !important;
}

  content: '' !important;
  display: block !important;

}

Custom Fonts

Google Fonts via @import

Works in OBS's embedded Chromium. Occasional flicker on first load while the SPA initialises. Always include a system font fallback.

@import url('https://fonts.googleapis.com/css2?family=Permanent+Marker&display=swap');

/* font-family stack: web font → bold system fallback → generic */
font-family: 'Permanent Marker', Impact, 'Arial Black', sans-serif !important;

Local Font via @font-face

More reliable if the Google Fonts import occasionally fails. Font must be installed on the machine running OBS.

@font-face {
  font-family: 'Permanent Marker';
  src: url('C:/Windows/Fonts/PermanentMarker-Regular.ttf');
}
Note: Courier New is a system font — do not attempt to load it from Google Fonts, it will be blocked by CORB.
